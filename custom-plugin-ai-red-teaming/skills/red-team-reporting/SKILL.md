# Red Team Reporting & Documentation

Professional **security report generation** for stakeholders, regulators, and internal teams with findings, remediation plans, and compliance evidence.

## Quick Start: Finding Document

```
Title: Prompt Injection Vulnerability
Severity: CRITICAL
Category: LLM Safety
CVSS Score: 9.8

Description:
LLM accepts and executes arbitrary instructions hidden in user input.

Impact:
- Generates harmful content bypassing safety training
- All API users affected
- Regulatory violation

Remediation:
1. Implement input validation (24h)
2. Harden system prompt (48h)
3. Deploy and test (72h)
4. Monitor (ongoing)

Status: IN PROGRESS
Owner: Security Team
Due: 2024-01-22
```

## Report Sections

1. **Executive Summary** (1 page)
   - Findings overview
   - Risk level
   - Remediation urgency

2. **Detailed Findings** (N pages)
   - Each vulnerability documented
   - POC and impact
   - Technical remediation

3. **Methodology** (1-2 pages)
   - Testing approach
   - Agents and techniques used
   - Limitations

4. **Roadmap** (1 page)
   - Immediate actions
   - Timeline
   - Resources

5. **Appendices**
   - Test cases
   - Payloads (sanitized)
   - References

## Documentation Quality Checklist

- ☐ Clear, non-technical for executives
- ☐ Technical details for engineers
- ☐ Business impact explained
- ☐ Remediation steps clear
- ☐ Timeline realistic
- ☐ Compliance mapping shown
- ☐ Evidence/proof documented
- ☐ Professional formatting
